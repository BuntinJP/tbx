# AGENTS.md

このリポジトリ（`~/.tbx`）を操作するAIエージェントへの指示書。

---

## リポジトリの目的

`~/.tbx` は個人用ワークフロースクリプト群。  
シェル操作・日常作業・開発補助のための小さなツール、設定、スクリプトを集約する個人用シェルツールボックス。

`chat-memo.txt` と `comment.sh` は、このリポジトリそのものの主目的ではなく、複数のAIエージェント・人間・自動化が同じ作業場で連携するための共有メモ基盤として使う。

---

## エージェントが必ず守るルール

### 1. コメント投稿は必ず `comment.sh` を使うこと

作業の開始・完了・エラー・判断・質問など、あらゆるステータス変化は `comment.sh` で `chat-memo.txt` に記録する。  
直接 `chat-memo.txt` をエディタで編集することは**禁止**。

```sh
./comment.sh "メッセージ" "agent-name"
```

### 2. エージェント名は実行主体がわかる名前にすること

`chat-memo.txt` は、複数のAIエージェント・人間・自動化が同じ場所に足跡を残すためのログ。  
したがって、エージェント名は単一名に固定しない。誰が・どの系統のツールが発言したかが後から読めるように、実行主体に合う名前を選ぶ。

同じ作業セッション内では同じ名前を使い、別のエージェントや別の実行環境から投稿するときは、その主体に合う別名を使ってよい。

| エージェント名 | 使用者 |
|---|---|
| `system` | 自動化スクリプト・CIなど無名プロセス（デフォルト） |
| `codex` | AIコーディングアシスタント（Codex） |
| `antigravity` | AIコーディングアシスタント（Antigravity） |
| `claude` | Claude 系アシスタント |
| `gemini` | Gemini 系アシスタント |
| `gpt` | GPT 系アシスタント全般 |
| `reviewer` | レビュー担当のAIまたは人間 |
| `planner` | 計画・設計担当のAIまたは人間 |
| `operator` | 手元操作・実行担当のAIまたは人間 |
| `user` | 人間（手動投稿時） |
| 任意の文字列 | 目的・人格・作業分担に応じて自由に定義してよい |

名前は「統制」よりも「読み返したときの面白さ・判別しやすさ」を優先してよい。  
ただし、誰の投稿かわからなくなる省略や、同一セッション内で意味なく名前を変えることは避ける。

### 3. 作業前後に必ず投稿すること

```sh
# 作業開始時
./comment.sh "Starting: <タスク概要>" "<agent-name>"

# 作業完了時
./comment.sh "Done: <完了内容>" "<agent-name>"

# エラー・ブロック時
./comment.sh "Blocked: <問題の説明>" "<agent-name>"
```

### 4. `chat-memo.txt` をこまめに確認すること

作業開始時、作業の区切り、判断に迷ったとき、他のエージェントの作業と衝突しそうなときは、`chat-memo.txt` を確認する。  
他のエージェント・人間・自動化の最新状況を踏まえ、重複作業や矛盾した変更を避ける。

`chat-memo.txt` を直接編集してはいけない。確認は `tail` や `less` などで読み取り専用に行い、投稿は必ず `comment.sh` を使う。

### 5. 他のエージェントにタスクを振るときは明言すること

他のエージェントや人間に作業を依頼・引き継ぎ・分担するときは、`comment.sh` で明示的に投稿する。  
誰に、何を、どこまで任せたのかが後から読めるように、担当者・対象ファイル・期待する成果・自分が継続する作業を簡潔に書く。

```sh
./comment.sh "Assign: @reviewer に scripts/foo.ts の挙動確認を依頼。@codex は AGENTS.md の更新を継続" "codex"
```

## `comment.sh` のシグネチャ

```sh
./comment.sh "comment content" [agent-name]
```

- `comment content` — 投稿するテキスト（必須）
- `agent-name` — 著者ラベル（省略時: `system`）

環境変数 `COMMENT_AGENT` でデフォルトエージェント名を設定できる。  
環境変数 `COMMENT_MEMO` で書き込み先ファイルを変更できる。

---

## 投稿フォーマット

`chat-memo.txt` への書き込み形式は以下の通り（スクリプトが自動生成する）：

```
────────────────────────────────────────────────────────────
[YYYY-MM-DDTHH:MM:SS+ZZZZ]  @agent-name

コメント本文
```

## スクリプト一覧

| スクリプト | パス | 説明 |
|---|---|---|
| `comment.sh` | `~/.tbx/comment.sh` | `chat-memo.txt` にコメントを追記する |
| `tbx_rc.sh` | `~/.tbx/tbx_rc.sh` | `bin` の PATH と shell alias を読み込む |

---

## 禁止事項

- `chat-memo.txt` を直接編集・削除・上書きしないこと
- `comment.sh` をバイパスしてメモに書き込まないこと
- エージェント名を省略して意図が不明瞭なエントリを作らないこと
